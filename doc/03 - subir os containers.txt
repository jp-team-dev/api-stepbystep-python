Suba os containers (se já subiu, pode pular):

    $ docker compose up -d
    ou
    $ make up

# Isso vai subir o banco de dados e o pgadmin.

# O pgadmin vai ficar disponível em http://localhost:8080, com usuário "

# Verifique se o banco de dados está rodando corretamente, acessando o pgadmin e conectando-se ao banco.

    1. Listar containers ativos

        $ docker ps

    2. Ver todos os containers (inclusive parados)

        $ docker ps -a

    3. Testar conexão direto no container

        $ docker exec -it db psql -U postgres -d stepbystep

    4. Logs do container

        $ docker logs db

    5. Remover containers parados

        $ docker rm $(docker ps -a -f status=exited -q)

    6. Parar os containers
    
        $ docker compose down

    7. Como remover containers no estado Created
    
        $ docker rm $(docker ps -a -f status=created -q)

        ou

        $ docker rm app-stepbystep-flutter-backend-1
        $ docker rm app-stepbystep-flutter-db-1
